01 ‚Äî HTML

Document Structure & Semantic Tags

<!-- Semantic tags tell browsers AND people what content means -->
<header>   <!-- Site-wide header: logo + nav -->
  <nav> ... </nav>
</header>

<main>             <!-- Primary page content (one per page) -->
  <article>       <!-- Self-contained content (blog post, project) -->
    <section> ... </section>
  </article>
  <aside> ... </aside>  <!-- Sidebar / related content -->
</main>

<footer> ... </footer>  <!-- Closing info: links, copyright -->

Headings

Use h1‚Äìh6 in order. Only one <h1> per page (your name or page title on your personal site).

<h1>Your Name</h1>

Links

Use <a href="..."> for navigation. Add target="_blank" rel="noopener" for external links.

<a href="/about">About</a>

Images

Always include alt text. Use loading="lazy" for images below the fold.

<img src="me.jpg" alt="Photo of me" loading="lazy">

Lists

<ul> for unordered (bullets), <ol> for numbered, <dl> for definitions.

<ul><li>Item</li></ul>

div vs span

<div> is a block container. <span> is an inline container. Use semantic tags when possible.

<span class="highlight">word</span>

Data Attributes

Store custom data on elements with data-*. Read in JS with el.dataset.key.

<div data-project="portfolio">

02 ‚Äî CSS

The Box Model

box-model.css 
.element {
  /* Content: width & height of the content area */
  width:  200px;
  height: 100px;

  /* Padding: space INSIDE the border */
  padding: 16px;           /* all sides */
  padding: 10px 20px;      /* top/bottom  left/right */

  /* Border: the line around padding + content */
  border: 2px solid #fff;

  /* Margin: space OUTSIDE the border */
  margin: 24px auto;    /* centres block horizontally */

  /* BEST PRACTICE: include border + padding in width */
  box-sizing: border-box;
}

The Document Object Model (DOM) is a JavaScript representation of your HTML. It lets you read and modify any element, attribute, or text on the page dynamically.
When you click a child element, the click event "bubbles" up through its parent elements too. Use e.stopPropagation() to stop it, or e.target to find exactly what was clicked.
Use const by default ‚Äî it prevents accidental reassignment. Switch to let only when you need to reassign the variable (like a counter). Never use var in modern JS.

09 ‚Äî Responsive Design

Media Queries & Mobile-First

responsive.css 
/* MOBILE-FIRST: write base styles for small screens,
   then EXPAND with min-width breakpoints */

.grid {
  display: grid;
  grid-template-columns: 1fr;          /* 1 col on mobile */
  gap: 16px;
}

@media (min-width: 768px) {             /* tablet+ */
  .grid { grid-template-columns: repeat(2, 1fr); }
}

@media (min-width: 1024px) {            /* desktop */
  .grid { grid-template-columns: repeat(3, 1fr); }
}

/* RESPONSIVE UNITS cheat sheet:
   rem  ‚Äî relative to root font size (16px default)
   em   ‚Äî relative to parent font size
   vw   ‚Äî % of viewport width
   vh   ‚Äî % of viewport height
   %    ‚Äî % of parent element
   clamp(min, preferred, max) ‚Äî responsive in one line */

Mobile-first

Write base styles for phones, then use min-width media queries to enhance for larger screens. Easier to maintain.

Fluid Images

Set max-width: 100% on all <img> tags so they never overflow their container.

clamp()

One CSS function to replace 3 breakpoints for font sizes. clamp(1rem, 3vw, 2rem) scales smoothly.

Viewport meta tag

Without <meta name="viewport" content="width=device-width, initial-scale=1">, mobile browsers will zoom out.
