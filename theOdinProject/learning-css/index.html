HTML Foundations

Boilerplate, tags/elements, text, lists, links, and images.

Introduction to HTML and CSS

HTML structures content. CSS styles it. Usually: HTML links to an external CSS file.

This page is HTML. Its look comes from CSS. The HTML file includes: <link rel="stylesheet" href="styles.css">

Inspect this element in DevTools → you’ll see its CSS rules.

HTML

<!-- In your HTML <head> -->
<link rel="stylesheet" href="styles.css" />

CSS

/* In styles.css */
.demo-note {
  border: 1px solid rgba(255,255,255,0.12);
  padding: 12px;
  border-radius: 12px;
}

HTML Boilerplate

Minimum structure for a modern HTML page.

You won’t “see” boilerplate on the page, but every page needs it.

Tip: Use ! + Tab in VS Code to generate boilerplate.

HTML

<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>My Page</title>
    <link rel="stylesheet" href="styles.css" />
  </head>
  <body>
    <h1>Hello</h1>
  </body>
</html>

How CSS gets applied, how conflicts resolve, and core layout rules.

Intro to CSS (3 ways)

Inline, internal (style tag), external (stylesheet). External is most common.

This line has a tiny inline style (border-bottom).

The rest of this site uses an external stylesheet: styles.css.

INLINE (HTML)

  Inline styled text

INTERNAL (HTML)

<style>
  .demo { color: hotpink; }
</style>

EXTERNAL (HTML)

<link rel="stylesheet" href="styles.css" />

The Cascade

When multiple rules apply, the browser uses specificity + order (later wins if tied).

This paragraph demonstrates the cascade.

In DevTools → “Styles” panel, you’ll see crossed-out rules losing to the winner.

HTML

<p class="cascade-demo">This paragraph demonstrates the cascade.</p>

CSS

/* Earlier rule */
.cascade-demo { color: var(--danger); }

/* Later rule (same specificity) wins */
.cascade-demo { color: var(--success); }

Growing and Shrinking

Use flex / flex-grow / flex-shrink to control sizing.

grow: 1
grow: 2 (bigger)
grow: 1

Middle item grows twice as much as the others.

CSS

.grow-1 { flex: 1; }
.grow-2 { flex: 2; }
What does flex: 1 mean?

flex: 1 is shorthand for grow/shrink/basis. In many cases it means “share remaining space equally.”
